This guide walks you through cloning the repository, opening it in Godot 4, and running your first match.

## Prerequisites

Before you begin, make sure you have:

- A supported OS: **Windows**, **Linux**, or **macOS**
- **Git** and a GitHub account
- [**Godot 4.6**](https://downloads.godotengine.org/?version=4.6&flavor=stable&slug=mono_win64.zip&platform=windows.64) (use the .NET/Mono build if you want C# support)
- [**Godot Export Templates**](https://downloads.godotengine.org/?version=4.6&flavor=stable&slug=mono_export_templates.tpz&platform=templates) (optional — only needed to export builds)

<Tip>
  If you only want to play or develop from the editor, you don't need the export templates. Godot 4.6 is enough to get started.
</Tip>

## Get the source

<Steps>
  <Step title="Clone the repository">
    Open a terminal in the folder where you want the project and run:

    ```bash
    git clone https://github.com/Andresit1524/beast_card_clash
    cd beast_card_clash
    ```
  </Step>

  <Step title="Open in Godot">
    Launch Godot 4.6 and use **Import** to open the `project.godot` file inside the cloned folder. Godot will import all assets automatically.

    <Note>
      The first import may take a minute as Godot processes textures, models, and dialogue files.
    </Note>
  </Step>

  <Step title="Run the game">
    Press **F5** (or click the **Play** button) in the Godot editor to launch the game from the main scene. The boot splash will appear, followed by the start menu.
  </Step>

  <Step title="Play your first match">
    From the start menu:
    1. Select your **character** (Bear, Condor, Chameleon, or Frog) and **skin**
    2. Choose your **team** (faculty)
    3. Enter the **battle** — roll the dice, place your character on a rock, and play cards each turn
    4. The **Referee** phase resolves battles after all turns are complete
    5. The last player standing wins
  </Step>
</Steps>
