This guide walks you through cloning the repository, opening the project in Godot 4, and configuring your editor. By the end you will have a working local copy of Beast Card Clash ready to run and modify.

## Prerequisites

Before you start, make sure you have the following installed:

<CardGroup cols={2}>
  <Card title="Godot 4.6" icon="gamepad" href="https://godotengine.org/download">
    The primary engine. Download the standard build — the .NET/Mono build is only needed if you work on C# systems.
  </Card>
  <Card title="Git" icon="code-branch" href="https://git-scm.com/downloads">
    Required to clone the repository and follow the contribution workflow.
  </Card>
  <Card title="GitHub account" icon="github" href="https://github.com">
    Needed to fork the repo, open pull requests, and collaborate.
  </Card>
  <Card title="VS Code (recommended)" icon="code" href="https://code.visualstudio.com">
    Recommended editor. Install the godot-tools extension for GDScript support.
  </Card>
</CardGroup>

<Note>
  The `.NET SDK 8+` and the Godot `.NET/Mono` build are **optional**. You only need them if you plan to work on the C# systems in the project. GDScript contributors do not need them.
</Note>

## Installation steps

<Steps>
  <Step title="Clone the repository">
    Open a terminal and run:

    ```bash
    git clone https://github.com/Andresit1524/beast_card_clash
    cd beast_card_clash
    ```

    This creates a local `beast_card_clash/` folder with the full project source.
  </Step>

  <Step title="Open the project in Godot">
    Launch **Godot 4.6**, click **Import** in the Project Manager, and navigate to the `beast_card_clash/` folder. Select the `project.godot` file and click **Import & Edit**.

    Godot will perform a first-run import — this may take a moment while it processes assets, generates `.godot/` cache files, and registers autoloads.

    <Tip>
      Wait for the import to finish fully before making any changes. Editing files while the importer is running can cause resource UID mismatches.
    </Tip>
  </Step>

  <Step title="Verify the project loads">
    Once the editor opens, confirm the following in the **FileSystem** and **Scene** panels:

    - The `project.godot` file lists version `0.1`.
    - The autoloads — `DialogueManager`, `SceneManager`, `MusicManager`, `FlagsManager`, `GameConstants`, `PlayerStats` — appear under **Project → Project Settings → Autoload**.
    - The plugins `dialogue_manager` and `vector_display_2d` are enabled under **Project → Project Settings → Plugins**.

    Press **F5** (or the Play button) to run the game and confirm the start menu appears.
  </Step>

  <Step title="Configure VS Code (recommended)">
    If you use VS Code as your editor:

    1. Install the [godot-tools](https://marketplace.visualstudio.com/items?itemName=geequlim.godot-tools) extension.
    2. Open the `beast_card_clash/` folder in VS Code.
    3. The `.vscode/` folder in the repository contains shared workspace settings — do **not** modify these files, as they keep editor configuration consistent across contributors.
    4. In Godot, go to **Editor → Editor Settings → Text Editor → External** and set VS Code as your external editor.

    <Warning>
      Do not edit or delete any file inside `.vscode/`. These settings are version-controlled and shared by all contributors. Override them only in your user-level VS Code settings if needed.
    </Warning>
  </Step>
</Steps>

## Platform notes

<Tabs>
  <Tab title="Windows">
    - Download the Godot 4.6 Windows executable (`.exe`) from the [official downloads page](https://godotengine.org/download).
    - Git is available via [Git for Windows](https://git-scm.com/download/win) or the [GitHub Desktop](https://desktop.github.com) installer.
    - Run all terminal commands in **Git Bash**, **PowerShell**, or **Windows Terminal** — plain `cmd.exe` works but is less ergonomic.
    - Godot does not require installation; you can run it directly from the extracted folder.
  </Tab>
  <Tab title="Linux">
    - Download the Godot 4.6 Linux binary from the [official downloads page](https://godotengine.org/download). Make it executable:

      ```bash
      chmod +x Godot_v4.6-stable_linux.x86_64
      ```

    - Git is available through your package manager:

      ```bash
      # Debian/Ubuntu
      sudo apt install git

      # Fedora
      sudo dnf install git

      # Arch
      sudo pacman -S git
      ```

    - You can optionally add the Godot binary to your `$PATH` for convenience.
  </Tab>
  <Tab title="macOS">
    - Download the Godot 4.6 macOS `.dmg` from the [official downloads page](https://godotengine.org/download) and drag it to your **Applications** folder.
    - macOS may quarantine the app on first launch. If you see a security warning, go to **System Settings → Privacy & Security** and click **Open Anyway**.
    - Install Git via [Homebrew](https://brew.sh):

      ```bash
      brew install git
      ```

    - Alternatively, installing Xcode Command Line Tools also provides Git:

      ```bash
      xcode-select --install
      ```
  </Tab>
</Tabs>

## Optional: C# / .NET setup

A small number of systems in the project support C#. You only need this setup if you are contributing to those systems.

<Steps>
  <Step title="Install the .NET SDK 8+">
    Download and install the [.NET SDK 8 or later](https://dotnet.microsoft.com/download) for your platform. Verify with:

    ```bash
    dotnet --version
    ```
  </Step>

  <Step title="Use the Godot .NET build">
    Download the **Godot 4.6 .NET** (Mono) build from the [official downloads page](https://godotengine.org/download). This is a separate executable from the standard build.
  </Step>

  <Step title="Open the project with the .NET build">
    Import the project using the .NET Godot executable. On first open, Godot will generate the C# project files (`.csproj`, `.sln`). These are already listed in `.gitignore` and should not be committed.
  </Step>
</Steps>

## Optional: Export templates

Export templates are required to build distributable binaries (e.g., a `.exe` for Windows or a web build). You do **not** need them to run or develop the game inside the Godot editor.

To install export templates:

1. In the Godot editor, go to **Editor → Manage Export Templates**.
2. Click **Download and Install** for Godot 4.6.
3. Wait for the download to complete.

Once installed, you can export builds via **Project → Export**.
